2) What is an Interpreted language?

An interpreted language is a programming language that does not require a separate compilation step before running the program. Instead, the program is executed line by line, with the interpreter interpreting and executing each line of code as it is encountered in the source code. Examples of interpreted languages include Python, Javascript, R, PHP, and Ruby. Since interpreted languages do not require a separate compilation step, they can be more flexible and easier to use than compiled languages, but may be slower in some cases.

5) What is namespace in Python? What are its types?

In Python, a namespace is a mapping between names (or identifiers) and objects. Namespaces are used to organize and control the visibility of names in a program.

In general, there are three types of namespaces in Python:

  1. Built-in Namespace: This namespace contains built-in functions and types provided by Python, which are always available for use in any Python program.
  2. Global Namespace: This namespace contains names defined at the module level. These names are accessible from any part of the module.
  3. Local Namespace: This namespace is created when a function is called and contains names defined within that function. These names are only accessible within the function and are not visible outside of it.

    4. When a name is referenced in a Python program, Python first searches the local namespace, then the global namespace, and finally the built-in namespace to resolve the name. This is known as the "LEGB" rule.

11) What are global variables and local variables?

In Python, global variables are variables that are defined outside of any function or class, and can be accessed and modified by any part of the program. Local variables, on the other hand, are variables that are defined within a function or class, and can only be accessed within that function or class.

Here is an example to illustrate the difference:

Example:
# Global variable
name = "John"

def print_name():
     # Local variable
     age = 25
     print("My name is", name)
     print("I am", age, "years old")
print_name()

In the above code, 'name' is a global variable that can be accessed within the 'print_name()' function. 'age', on the other hand, is a local variable that is defined only within the 'print_name()' function and cannot be accessed outside of it.

Example:
# Loacal variable
count = 0
def increment_count():
     global count
     count += 1
     increment_count()
print(count) # Output: 1

In the above code, the 'global' keyword is used to tell Python that 'count' refers to the global variable and not a local variable with the same name. The 'increment_count()' function then modifies the global variable 'count'.

12) What are the types of parameter passing mechanisms in Python?

In Python, there are two types of parameter passing mechanisms:

  1. Pass by reference: In Python, all variables are actually references to objects in memory. When a function is called and a parameter is passed by reference, the function receives a reference to the same object as the original variable. Any changes made to the parameter within the function are reflected in the original variable outside the function.
  2. Pass by value: In some cases, when a parameter is passed by value, a copy of the parameter's value is made and passed to the function. This means that any changes made to the parameter within the function do not affect the original variable outside the function.

It's important to note that even when a parameter is passed by value, if the value is an object (such as a list or dictionary), the object itself is still passed by reference. This means that changes made to the object within the function will be reflected in the original variable outside the function.

In Python, the default parameter passing mechanism is pass by reference. However, you can force a parameter to be passed by value by making a copy of the parameter using the copy module or by explicitly assigning a new value to the parameter within the function.

17) How does break and continue keywords work?

break and continue are the two keywords in Python that are used to control the flow of loops, such as for and while loops.

break is used to immediately terminate a loop, even if the loop's condition is still true. When break is encountered, the loop is exited and the program continues executing from the next statement after the loop. Here's an example:

Example:
for i in range(1, 11):
     if i == 5:
          break
print(i)

In this example, the loop will print out the numbers 1 through 4, but when i equals 5, the break statement is executed, and the loop is immediately terminated. The program then continues executing from the next statement after the loop, which is nothing in this case.

continue, on the other hand, is used to skip over the current iteration of a loop and move on to the next iteration. When continue is encountered, the remaining statements in the loop body for the current iteration are skipped, and the loop condition is evaluated again for the next iteration. Here's an example:

Example:
for i in range(1, 11):
     if i == 5:
     continue
     print(i)

In this example, the loop will print out all the numbers from 1 through 10, except for 5. When i equals 5, the continue statement is executed, which skips the remaining statements in the loop body for that iteration and moves on to the next iteration.

break and continue are often used together with conditional statements, such as if statements, to control the flow of a loop based on certain conditions. They can help make loops more efficient and reduce the complexity of the code.

18) What is slicing in Python?

Slicing is a technique in Python used to extract a portion of a sequence (such as a string, list, or tuple) by specifying a range of indices. In Python, we can slice sequences using the colon : operator inside square brackets [ ]. The general syntax for slicing is as follows:

Syntax: dataStructure[start:end:step]

where:

  • start: The index of the first element to be included in the slice (inclusive). If omitted, it defaults to 0.
  • end: The index of the last element to be included in the slice (exclusive). If omitted, it defaults to the length of the sequence.
  • step: The number of indices to skip between each element in the slice. If omitted, it defaults to 1.
Here are some examples of using slicing in Python:

Example:
my_string = "Hello, World!"
my_list = [1, 2, 3, 4, 5]

# Slicing a string
print(my_string[0:5]) # Output: "Hello"
print(my_string[7:]) # Output: "World!"
print(my_string[::2]) # Output: "Hlo ol!"

# Slicing a list print(my_list[1:3]) # Output: [2, 3]
print(my_list[:-1]) # Output: [1, 2, 3, 4]
print(my_list[::2]) # Output: [1, 3, 5]

19) What does [::-1] do?

In Python, [::-1] is a slicing notation used on a sequence, such as a list or a string, to reverse the order of its elements. The notation [start:stop:step] is used to slice a sequence from index start to index stop with a step of step.

When the start and stop indices are not specified, the entire sequence is considered, and when the step is negative, the sequence is traversed in reverse order. So, [:: -1] means to start at the beginning of the sequence, end at the end of the sequence (implicitly), and traverse the sequence backwards with a step of -1. Below is an example of using [::-1] to reverse a list

Example:
myList = [1, 2, 3, 4, 5]
myReversedList = myList [::-1]
print(myReversedList) # Output: [5, 4, 3, 2, 1]

In this example, the original list ' [1, 2, 3, 4, 5] ' is reversed using [::-1], and the resulting reversed list is ' [5, 4, 3, 2, 1] '. This slicing notation is commonly used when we need to reverse a list or a string in Python.

36) What is the difference between set and dictionary?

Both sets and dictionaries are built-in data types in Python, but they have some differences:

  1. Purpose: The main purpose of a set is to store a collection of unique elements, while a dictionary is used to store key-value pairs, where each key has an associated value
  2. Syntax: A set is defined using curly braces {} with comma-separated values, while a dictionary is defined using curly braces {} with comma-separated key-value pairs in the form of key: value.
  3. Indexing: A set is an unordered collection, so its elements cannot be accessed using an index. On the other hand, a dictionary is indexed using its keys to retrieve their corresponding values.
  4. Mutability: Sets are mutable, which means that you can add or remove elements from them. Dictionaries are also mutable, but you can only modify their values, not their keys.
  5. Duplicates: Sets don't allow duplicates, while dictionaries can have duplicate values, but not duplicate keys
Here's an example code snippet to illustrate the difference between sets and dictionaries:

Example:
# Example of a set
my_set = {1, 2, 3, 4, 5, 1, 2}
print(my_set) # Output: {1, 2, 3, 4, 5}

# Example of a dictionary
my_dict = {'apple': 2, 'banana': 3, 'orange': 4}
print(my_dict) # Output: {'apple': 2, 'banana': 3, 'orange': 4}

In the example above, the set my_set only contains unique elements, while the dictionary my_dict contains key-value pairs.

The "for" loop iterates through each number in the range, and the square of each number is added to the list using the append() method.

44) What are the differences between zip(), map(), filter() ?

  1. zip(): The zip() function takes iterables as input and returns an iterator of tuples. The returned tuples contain elements from each input iterable, with the i-th tuple containing the i-th element from each iterable

    Example:
    a = [1, 2, 3]
    b = ['one', 'two', 'three']
    zipped = zip(a, b)
    print(list(zipped)) # Output: [(1, 'one'), (2, 'two'), (3, 'three')]

  2. map(): The map() function takes a function and one or more iterables as input and returns an iterator. It applies the given function to each element of the iterables and returns an iterator of the results.

    Example:
    def square(x):
         return x*x
    a = [1, 2, 3, 4]
    squared = map(square, a)
    print(list(squared)) # Output: [1, 4, 9, 16]

  3. filter(): The filter() function takes a function and an iterable as input and returns an iterator of elements for which the function returns True. It filters out elements from the iterable that do not meet the condition specified in the function.

    Example:
    def is_even(x):
         return x % 2 == 0
    a = [1, 2, 3, 4, 5, 6]
    even_nums = filter(is_even, a)
    print(list(even_nums)) # Output: [2, 4, 6]

The main differences between these three functions are as follows:

  • zip() is used to combine two or more iterables into a single iterable of tuples.
  • map() is used to apply a function to each element of an iterable and return an iterator of the results.
  • filter() is used to apply a function to each element of an iterable and return an iterator of the elements for which the function returns True.
Each function serves a different purpose, and their usage depends on the specific needs of the program.

45) How exception handling is done in Python?

Exception handling is a way to handle errors or exceptions that occur during program execution. In Python, exceptions are raised when an error or unexpected situation occurs during program execution.

To handle exceptions in Python, we use the try and except blocks. The code that may raise an exception is written inside the try block. If an exception is raised during the execution of the code inside the try block, the program control is transferred to the except block where we can handle the exception.

We can also use the finally block to ensure that certain code is executed regardless of whether an exception is raised or not. The finally block is executed after the try and except blocks. Here is an example:

Example:
try:
     # code that may raise an exception
     x = int(input("Enter a number: "))
     y = 1/x
     print(y)
except ZeroDivisionError:
     print("Cannot divide by zero")
except ValueError:
     print("Invalid input. Please enter a valid number.")
finally:
     print("Program execution complete")

In the above example, we are trying to divide 1 by the number entered by the user. We use the try block to handle any exceptions that may occur during the execution of this code. If the user enters 0, a ZeroDivisionError is raised. If the user enters a non-numeric value, a ValueError is raised. We handle these exceptions separately in the except blocks. "Program execution complete" is printed after the try and except blocks have executed, regardless of whether an exception was raised or not.

48) Does Python support multiple inheritance?

Yes, Python supports multiple inheritance, which means that a class can inherit attributes and methods from multiple parent classes. In Python, a class can be derived from more than one base class, by specifying multiple base classes in the class definition using a comma-separated list of parent classes. Here's an example:

Example:
class A:
     def method_A(self):
          print("Method A")

class B:
     def method_B(self):
          print("Method B")

class C(A, B):
     def method_C(self):
          print("Method C")

In the above example, the class "C" is derived from both "A" and "B" classes. Therefore, "C" will inherit all the methods and attributes from both "A" and "B" classes, along with its own methods and attributes.
